Purpose of the framework

This performance framework was developed to get an idea of how ballerina performance changes with nightly builds and how ballerina performs compared to other languages in different use cases.

Test Execution
  1. Create a fresh single node cluster(Standard_F4s_v2) Kubernetes cluster in Azure and installs Nginx ingress.
  2. Deploy the application pod into the cluster (Hello world service written by different languages with 1 cpu, 512mb memory resource limits).
  3. Deploy the JMeter pod and execute the benchmark and then commit the results to Github. (Concurrent Users - 50)
  4. Cleanup the cluster.
Performance Accuracy

Due to infrastructure-related reasons, we have noticed about 10% variance in worst-case test execution. Additionally, we haven't optimized any application for optimal settings and only wrote a simple hello world as our objective is simply to get a rough idea of where we are. This is not an effort to get the most accurate numbers. You could view the source codes(and other artifacts) for scenarios here.
